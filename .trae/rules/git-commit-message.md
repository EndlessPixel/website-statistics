# Git Commit 提交信息规范规则
## 一、整体格式
采用 Angular 规范格式：`类型(可选范围): 简短描述`
可选追加换行详情块（复杂修改才加，小改动只一行）

## 二、固定可用类型
1. feat：新增功能、接口、页面、配置、脚本
2. fix：bug修复、逻辑异常、参数兼容问题
3. docs：README、文档、注释、文案修改
4. chore：依赖更新、gitignore、构建配置、清理测试代码
5. refactor：代码重构，无功能增减无bug修复
6. style：样式、图标、布局、css微调、格式美化
7. perf：性能优化、请求延时、数据库读写优化
8. test：填充测试数据、测试接口、调试脚本

## 三、范围scope填写规则
根据项目模块填写，无分模块可省略括号
- stat：统计核心上报逻辑
- ui：前端页面、图表、弹窗、主题
- auth：登录密钥、权限、域名白名单
- deploy：Docker、部署脚本、pm2配置
- db：SQLite建表、字段、存储逻辑
- proxy：反向代理IP解析逻辑
- doc：所有md文档

## 四、文字书写要求
1. 简短描述使用英文小写动词开头，语态祈使句
2. 长度控制50字符内，不写长句
3. 详情块逐条分点，清晰说明改动内容、原因、影响
4. 禁止模糊描述：`更新代码`、`改了点东西`、`优化一下`
5. 涉及配置/文件修改可以在详情标注文件名
6. 中文项目全程英文commit，保持开源仓库通用可读性

## 五、特殊场景约束
1. 临时测试填充数据脚本：统一type=test
2. 删演示接口/脏代码：chore
3. README大段重写：docs
4. 修复数据库写入报错、锁冲突：fix(db)
5. 新增埋点示例、Nginx/Caddy配置：feat(doc)
6. 调整页面深浅色、图标替换：style(ui)

# 六、其他约束
1. 必须使用中文填写提交信息，保持项目代码库的中文可读性。
